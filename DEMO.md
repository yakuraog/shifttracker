# Как протестировать ShiftTracker

## Что это

Я разработал систему, которая автоматически заполняет таблицу смен по фотографиям из Telegram-групп. Охранник просто кидает фото в рабочий чат при заступлении на смену — система сама определяет кто это и ставит отметку "1" в таблице.

## Что ещё можно проверить

### Спорные случаи
Отправьте фото **без подписи** или с подписью которая не совпадает ни с одним сотрудником. Запись попадёт в **Очередь проверки** — там оператор может вручную подтвердить или отклонить.

### Защита от дублей
Отправьте второе фото с подписью `Иванов` в тот же день. Система не создаст вторую отметку — дубликат будет залогирован.

### Управление справочниками
В админке можно добавлять/редактировать/удалять группы, сотрудников, правила подписей. Всё через веб-интерфейс.

### Дашборд
На главной странице — статистика: сколько принято сегодня, сколько ожидает проверки, ошибки. Плюс график обработки за неделю.

### API документация
Swagger UI доступен по адресу: https://shifttracker-1ve0.onrender.com/docs

## Исходный код

GitHub: https://github.com/yakuraog/shifttracker

- 114 автоматических тестов
- Python 3.11, aiogram 3, FastAPI, SQLAlchemy 2
- Чистая архитектура: pipeline из 5 стадий обработки
- Готовность к масштабированию: asyncio + 8 воркеров + очередь с backpressure
- Google Sheets интеграция (батчинг, retry, защита от дублей)

## Архитектура (кратко)

```
Telegram группа → Бот → Очередь → Воркеры → БД → Google Sheets
                                     ↓
                              Админка (FastAPI)
```

Сотрудник отправляет фото → бот принимает → система определяет кто это (по подписи или Telegram-аккаунту) → определяет дату смены → ставит "1" в таблицу. Если определить не получилось — отправляет на ручную проверку оператору.
