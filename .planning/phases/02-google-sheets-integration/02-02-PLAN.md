   ```python
   class SheetsWriter:
       def __init__(self, settings: Settings, session_factory):
           self._settings = settings
           self._session_factory = session_factory
           self._task: asyncio.Task | None = None
           self._gc: gspread.Client | None = None

       async def start(self) -> None:
           if not self._settings.google_sheets_credentials_file:
               logger.warning("GOOGLE_SHEETS_CREDENTIALS_FILE not set, SheetsWriter disabled")
               return
           # Validate credentials file exists at startup (fail fast)
           cred_path = self._settings.google_sheets_credentials_file
           if not os.path.exists(cred_path):
               raise RuntimeError(f"GOOGLE_SHEETS_CREDENTIALS_FILE not found: {cred_path}")
           self._gc = await _run_sync(build_client, cred_path)
           self._task = asyncio.create_task(self._flush_loop())
           logger.info("SheetsWriter background task started")

       async def stop(self) -> None:
           if self._task:
               self._task.cancel()
               try: await self._task
               except asyncio.CancelledError: pass

       async def _flush_loop(self) -> None:
           while True:
               await asyncio.sleep(self._settings.sheets_flush_interval)
               try:
                   await self._flush()
               except Exception as e:
                   logger.error(f"SheetsWriter flush error: {e}")

       async def _flush(self) -> None:
           # Implementation steps:
           # a) Query all ShiftRecord with sheet_write_status="PENDING"
           #    JOIN with GroupEmployee (on employee_id + group linkage) to get sheet_row
           #    JOIN with TelegramGroup to get sheet_id, sheet_name
           # b) Skip records where TelegramGroup.sheet_id is None (log and continue)
           # c) Skip records where sheet_row is None or 0 (set ERROR + reason "employee_row_not_configured")
           # d) Group remaining records by (sheet_id, sheet_name)
           # e) For each group:
           #    - Get or refresh header row (use header_cache)
           #    - For each record, call resolve_cell(header_row, sheet_row, shift_date)
           #    - If resolve returns None: set ERROR + reason "date_column_not_found"
           #    - Collect resolved cells for batch_get (duplicate check)
           #    - Read existing cell values with batch_get in one call
           #    - For cells already containing "1": skip, create ProcessingLog with status="DUPLICATE_SHEET_SKIP"
           #    - Build batch_update payload with remaining cells: [{"range": value_a1, "values": [["1"]]}, {"range": link_a1, "values": [[source_link]]}]
           #    - Call worksheet.batch_update via _run_sync
           #    - On success: update each record sheet_write_status="WRITTEN", written_at=now
           #    - On gspread.exceptions.APIError: increment retry_count
           #      - If retry_count >= settings.sheets_max_retries: set status="ERROR"
           #      - Otherwise: keep status="PENDING" for next cycle
   ```

   The `_run_sync` helper:
   ```python
   async def _run_sync(fn, *args, **kwargs):
       loop = asyncio.get_event_loop()
       return await loop.run_in_executor(None, functools.partial(fn, *args, **kwargs))
   ```

   **Important details:**
   - The query for PENDING records needs to join ShiftRecord -> Employee -> GroupEmployee -> TelegramGroup to resolve sheet_id and sheet_row. Since an employee can belong to multiple groups, use the group from which the photo was received. The ShiftRecord does NOT have a group_id column directly. Look at how source_link contains chat_id info, or check if ProcessingLog has chat_id. The ProcessingLog has chat_id — use it to find the TelegramGroup. OR: add a subquery that finds the TelegramGroup via ProcessingLog.chat_id matching TelegramGroup.chat_id for the same message_id. Alternatively, look at what data is available: ShiftRecord has source_message_id but no chat_id. The simplest correct approach: query ProcessingLog for the same source_message_id to get chat_id, then match to TelegramGroup. Cache this mapping since it won't change.
   - Actually, a simpler approach: since each employee belongs to groups via GroupEmployee, and each group has a sheet_id, query GroupEmployee joined with TelegramGroup. If an employee is in exactly one group with a sheet_id configured, use that. If multiple groups, use the ProcessingLog.chat_id to disambiguate. For the MVP, if the employee is in exactly one group with a sheet configured, use it. Log ambiguity as an error.
   - DB updates must be committed within the same flush to ensure consistency (either WRITTEN or retry_count incremented). Never leave a record as PENDING after a successful write.
