    - Test: ShiftRecord with status REJECTED does NOT appear in the grid

  <action>
    1. Create shifttracker/admin/routers/shifts.py:
       - `router = APIRouter(prefix="/shifts", tags=["shifts"])`

       - GET "/" — shift grid view:
         - Accept query params: group_id (uuid, optional), date_from (date, optional), date_to (date, optional).
         - Default date range: today minus 6 days to today (7-day window).
         - Max date range: 31 days (clamp if wider).
         - Load all TelegramGroup for the group filter dropdown.
         - If group_id provided:
           a. Query GroupEmployee WHERE group_id, ordered by sheet_row (or employee name). Use selectinload for employee relationship.
           b. Collect employee_ids from the group.
           c. Query ShiftRecord WHERE employee_id IN employee_ids AND shift_date BETWEEN date_from AND date_to AND status IN ("CONFIRMED", "ACCEPTED").
           d. Build grid dict: `{(employee_id, shift_date): shift_record}`.
           e. Build date columns list: `[date_from, date_from+1day, ..., date_to]` using `timedelta(days=1)` loop.
           f. Pass to template: employees list, date_columns list, grid dict, groups (for filter dropdown), current filters.
         - If no group_id: show empty grid with only the filter form. Or show all employees across all groups (simpler: require group selection).

       - GET "/{record_id}/detail" (optional, for drill-down):
         - Load ShiftRecord by id.
         - Load related ProcessingLog entries for same employee_id + shift_date.
         - Return a partial HTML fragment showing: source_link, processing history entries with status/reason/created_at.
         - Used by htmx: clicking a "1" cell triggers hx-get to load detail in a popover or below the table.

    2. Update shifttracker/admin/router.py — include shifts router with require_session dependency.

    3. Create shifttracker/templates/admin/shifts/grid.html:
       - Extends base.html.
       - Filter form: group dropdown (required), date_from, date_to (date inputs). Submit button.
       - If no group selected: show message "Select a group to view the shift table".
       - If group selected: render HTML table.
         - Header row: "Employee" + one column per date in date_columns (formatted as DD.MM).
         - Each row: employee.name in first column, then for each date: if (employee.id, date) in grid: show "1" as a link/button with hx-get="/admin/shifts/{record.id}/detail" hx-target="#detail-panel" for drill-down. Else: empty cell.
       - Detail panel: a div below the table (`id="detail-panel"`) where htmx loads shift record details.
       - Style: Bootstrap table-striped, table-bordered. "1" cells highlighted with bg-success-subtle or similar.

    4. Create tests/test_admin_shifts.py:
       - test_shift_grid_query: seed a TelegramGroup, two Employees, GroupEmployee bindings, and ShiftRecords (one ACCEPTED, one REJECTED). Login. GET /admin/shifts/?group_id=X&date_from=Y&date_to=Z. Assert the ACCEPTED employee's name and "1" appear. Assert the REJECTED one does NOT show "1".
       - test_shift_grid_default_date_range: GET /admin/shifts/?group_id=X without date params. Assert 200. Assert the grid renders (default 7-day range).
       - test_shift_grid_requires_group: GET /admin/shifts/ without group_id. Assert 200 with "Select a group" message.
