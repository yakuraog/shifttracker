Implement the manual review queue: list NEEDS_REVIEW items, approve (create ShiftRecord with PENDING Sheets write), and reject (with operator comment).

Purpose: When the bot cannot identify an employee, the message goes to NEEDS_REVIEW. Operators use this queue to manually resolve these items — completing the human-in-the-loop workflow.
Output: Working review queue page with approve/reject actions, all backed by automated tests.
</objective>

  <action>
    1. Create shifttracker/admin/routers/review.py:
       - `router = APIRouter(prefix="/review", tags=["review"])`

       - GET "/" — list view:
         - Query ProcessingLog WHERE status = "NEEDS_REVIEW", ordered by created_at DESC.
         - For each entry, resolve employee name (LEFT JOIN Employee on employee_id) and group name (LEFT JOIN TelegramGroup on chat_id).
         - Accept optional query params: group_id (uuid), date_from (date), date_to (date) for filtering.
         - If group_id provided: filter WHERE chat_id = (SELECT chat_id FROM TelegramGroup WHERE id = group_id).
         - If date_from/date_to: filter WHERE created_at between dates.
         - Also load all TelegramGroup and Employee lists for the approve form dropdowns.
         - Render admin/review/list.html with items, groups, employees, current filters.

       - POST "/{log_id}/approve":
         - Accept Form: employee_id (uuid), shift_date (date).
         - Load ProcessingLog by id, 404 if not found or status != "NEEDS_REVIEW".
         - Check for existing ShiftRecord with same employee_id + shift_date. If exists, return 409 Conflict (or render error message for htmx).
         - Create ShiftRecord: employee_id, shift_date, status="ACCEPTED", source_message_id=log.message_id, source_link=log.source_link or "", sheet_write_status="PENDING".
         - Update log: status="ACCEPTED", employee_id=employee_id, shift_date=shift_date.
         - Commit.
         - Return Response(status_code=200) with empty body for htmx row removal. OR return HX-Redirect header to refresh the page.

       - POST "/{log_id}/reject":
         - Accept Form: comment (str).
         - Load ProcessingLog by id, 404 if not found or status != "NEEDS_REVIEW".
         - Update: status="REJECTED", reason=comment.
         - Commit.
         - Return Response(status_code=200) for htmx row removal.

    2. Update shifttracker/admin/router.py — include review router with require_session dependency.

    3. Create shifttracker/templates/admin/review/list.html:
       - Extends base.html.
       - Filter form at top: group dropdown (all groups + "All"), date_from, date_to inputs. Submit reloads the page with query params.
       - Table: columns (Created, Group, Employee, Reason, Source, Actions).
       - Each row shows: created_at formatted, group name (or chat_id if no group), employee name (or "Unknown"), reason text, source_link as a clickable link opening in new tab.
       - Actions column: Approve form (inline or modal) with employee_id dropdown and shift_date input, and a Reject form with comment textarea.
       - For Approve: use hx-post="/admin/review/{id}/approve" with hx-target="closest tr" hx-swap="outerHTML" — the row disappears on success.
       - For Reject: use hx-post="/admin/review/{id}/reject" similarly.
       - Consider using Bootstrap collapse or accordion for the approve/reject forms inline.

    4. Create shifttracker/templates/admin/review/_item.html:
       - Partial template for a single review queue row. Used if htmx needs to swap individual rows.
